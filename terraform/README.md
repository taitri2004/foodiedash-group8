# FoodieDash Terraform Infrastructure

## Cấu trúc file

```
foodiedash-terraform/
├── main.tf                  ← Provider + backend config
├── variables.tf             ← Tất cả variables
├── vpc.tf                   ← VPC, Subnets, Security Groups, NAT GW
├── docdb.tf                 ← DocumentDB cluster + instances
├── ecs.tf                   ← ECR, IAM, ECS Cluster + Service, ALB
├── s3_cloudfront.tf         ← S3 buckets + CloudFront distribution
├── apigateway_lambda.tf     ← API Gateway, VPC Link, Lambda functions
├── outputs.tf               ← Output values sau deploy
├── package_lambdas.sh       ← Script zip Lambda code
└── lambda/
    ├── ai_handler/
    │   └── index.js         ← Bedrock RAG handler code
    ├── sync_kb/
    │   └── index.js         ← KB sync handler code
    ├── ai_handler.zip       ← (generated by package_lambdas.sh)
    └── sync_kb.zip          ← (generated by package_lambdas.sh)
```

## Trước khi deploy

### 1. Cài đặt Terraform

```bash
# Windows (Chocolatey)
choco install terraform

# hoặc download từ https://developer.hashicorp.com/terraform/downloads
```

### 2. Điền secrets vào variables.tf

Mở `variables.tf` và thay tất cả `REPLACE_ME`:

```hcl
variable "app_secrets" {
  default = {
    MONGODB_URI     = "mongodb://foodiedashAdmin:PASSWORD@..."
    AUTH_JWT_SECRET = "your-super-secret-key"
    # ...
  }
}

variable "docdb_master_password" {
  default = "YourSecurePassword123"
}
```

Hoặc tạo file `terraform.tfvars` (không commit lên git):

```hcl
docdb_master_password = "YourSecurePassword123"
app_secrets = {
  MONGODB_URI            = "mongodb://..."
  AUTH_JWT_SECRET        = "..."
  AUTH_JWT_REFRESH_SECRET = "..."
  CLOUDINARY_CLOUD_NAME  = "..."
  CLOUDINARY_API_KEY     = "..."
  CLOUDINARY_API_SECRET  = "..."
  GEMINI_API_KEY         = "..."
  GROQ_API_KEY           = "..."
  APP_ORIGIN             = "https://REPLACE_AFTER_DEPLOY"
  PAYOS_CLIENT_ID        = "..."
  PAYOS_API_KEY          = "..."
  PAYOS_CHECKSUM_KEY     = "..."
  GOOGLE_APP_USER        = "..."
  GOOGLE_APP_PASSWORD    = "..."
  PORT                   = "4000"
  NODE_ENV               = "production"
  S3_BUCKET_MEDIA        = "foodiedash-s3-media"
}
```

### 3. Package Lambda functions

```bash
# Linux/Mac
chmod +x package_lambdas.sh
./package_lambdas.sh

# Windows (PowerShell)
cd lambda/ai_handler; Compress-Archive -Path index.js -DestinationPath ../../lambda/ai_handler.zip; cd ../..
cd lambda/sync_kb; Compress-Archive -Path index.js -DestinationPath ../../lambda/sync_kb.zip; cd ../..
```

### 4. Configure AWS credentials

```bash
aws configure
# Nhập: Access Key, Secret Key, Region: us-west-2
```

## Deploy

```bash
# Bước 1: Initialize
terraform init

# Bước 2: Preview
terraform plan

# Bước 3: Deploy
terraform apply
# Nhập 'yes' để confirm

# Sau deploy, copy output CloudFront domain
```

## Sau deploy — checklist

### Bước 1: Update APP_ORIGIN
Copy `cloudfront_domain` từ output → vào Secrets Manager → sửa `APP_ORIGIN`

### Bước 2: Update CORS
Sửa `variables.tf`:
```hcl
variable "cloudfront_domain" {
  default = "d1234567890.cloudfront.net"  # domain thật từ output
}
```
→ `terraform apply` lại để update CORS config

### Bước 3: Push Docker image
```bash
# Login ECR
aws ecr get-login-password --region us-west-2 | docker login --username AWS --password-stdin 490178243777.dkr.ecr.us-west-2.amazonaws.com

# Build và push
cd XOPS_BE
docker build -t foodiedash-be .
docker tag foodiedash-be:latest 490178243777.dkr.ecr.us-west-2.amazonaws.com/foodiedash-be:latest
docker push 490178243777.dkr.ecr.us-west-2.amazonaws.com/foodiedash-be:latest
```

### Bước 4: Deploy FE
```bash
cd XOPS_FE
npm run build
aws s3 sync dist/ s3://foodiedash-fe-490178243777 --delete --region us-west-2

# Lấy distribution ID từ output cloudfront_distribution_id
aws cloudfront create-invalidation --distribution-id DIST_ID --paths "/*" --region us-west-2
```

### Bước 5: Tạo Bedrock Knowledge Base (thủ công)
1. AWS Console → Amazon Bedrock → Knowledge Bases → Create
2. Data source: S3 → `foodie-knowledgebase`
3. Embedding: Titan Embeddings G1
4. Vector store: OpenSearch Serverless
5. Copy KB ID và DS ID

### Bước 6: Update Lambda env vars
```bash
# Update KB ID
aws lambda update-function-configuration \
  --function-name foodiedash-ai-handler \
  --environment "Variables={KNOWLEDGE_BASE_ID=YOUR_KB_ID}" \
  --region us-west-2

aws lambda update-function-configuration \
  --function-name foodiedash-sync-kb \
  --environment "Variables={KNOWLEDGE_BASE_ID=YOUR_KB_ID,DATA_SOURCE_ID=YOUR_DS_ID,API_URL=https://YOUR_APIGW_ID.execute-api.us-west-2.amazonaws.com,API_TOKEN=YOUR_ADMIN_TOKEN}" \
  --region us-west-2
```

### Bước 7: Upload KB files và sync
```bash
aws s3 cp ingredients.json s3://foodie-knowledgebase/
aws s3 cp menu_descriptions.txt s3://foodie-knowledgebase/
aws s3 cp allergen_guidelines.txt s3://foodie-knowledgebase/

# Trigger sync thủ công
aws lambda invoke --function-name foodiedash-sync-kb --region us-west-2 response.json
```

### Bước 8: DMS Migration (thủ công)
Tạo DMS task qua AWS Console: Atlas MongoDB → DocumentDB

## Destroy (xóa toàn bộ)

```bash
terraform destroy
```

⚠️ **Lưu ý:** ECR repository và DocumentDB cluster giữ data — xóa thủ công nếu cần.

## Lưu ý kỹ thuật

- DocumentDB chỉ support **SCRAM-SHA-1**, không hỗ trợ SCRAM-SHA-256
- Connection string phải có `retryWrites=false` và `tlsCAFile=global-bundle.pem`
- Lambda AI handler timeout 30s — đủ cho Bedrock RAG response
- EventBridge chạy lúc 19:00 UTC = 02:00 AM Vietnam time
